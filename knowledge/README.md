# Base de Conhecimento - Robo CEF

## Objetivo

Centralizar conhecimento funcional, tecnico e operacional do Robo CEF para consulta por desenvolvimento, QA, suporte, atendimento ao cliente e treinamento.

## Fontes

- `AGENTS.md`
- `prompts/04-gerar-conhecimento.md`
- `outputs/relatorios/inventario-projeto.md`
- `docs/funcional/*.md`
- `docs/tecnica/*.md`
- `docs/qa/cenarios-teste.md`
- `docs/suporte/perguntas-frequentes.md`
- `Robo-CEF/Program.cs`
- `Robo-CEF/Services/*.cs`
- `Robo-CEF/Repositories/*.cs`
- `Robo-CEF/Models/*.cs`
- `Robo-CEF/Constants/*.cs`

## Mapa

| Area | Conteudo |
|---|---|
| `funcionalidades/` | Funcionalidades e fluxos de negocio do robo |
| `arquitetura/` | Visao arquitetural e organizacao tecnica |
| `backend/` | Modulos backend e responsabilidades |
| `frontend/` | Situacao de frontend/telas proprias |
| `banco/` | Objetos de banco citados pelo codigo |
| `regras-negocio/` | Regras comprovadas no codigo |
| `integrações/` | Integracoes externas e dependencias |
| `faq/` | Perguntas recorrentes para suporte e atendimento |
| `bugs-conhecidos/` | Riscos, limitacoes e pontos a validar |
| `releases/` | Historico de atualizacoes da base |
| `cliente/` | Resumo em linguagem de atendimento |
| `treinamento/` | Guia inicial de operacao e entendimento |

## Politica de manutencao

Todo conhecimento deve conter origem da informacao. Quando algo nao estiver comprovado no codigo ou na documentacao existente, deve ser marcado como "Ponto a validar".
