# Responder cliente

## Objetivo
Responder dúvidas do cliente usando somente informações oficiais do projeto.

## Antes de responder
Consultar nesta ordem:

1. .memory/
2. knowledge/
3. docs/
4. tickets/
5. outputs/

## Regras
- Não inventar informação.
- Não expor código, senha, token ou dado sensível.
- Se não houver informação suficiente, responder que será necessário validar com o time técnico.
- Responder de forma clara, objetiva e profissional.
- Se a dúvida indicar erro funcional, sugerir abertura de chamado.
- Usar o template `/templates/resposta-cliente.md`.

## Classificação da resposta
Identifique se a solicitação é:

- Dúvida
- Bug
- Melhoria
- Incidente
- Solicitação operacional

## Saída
Salvar a resposta em:

support/respostas/RESPOSTA-AAAA-MM-DD-[resumo-curto].md

## Estrutura obrigatória
- Cliente
- Solicitação recebida
- Classificação
- Resposta sugerida
- Base consultada
- Segurança da resposta: Alta, Média ou Baixa
- Próximo passo
