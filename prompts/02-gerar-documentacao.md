# Gerar Documentacao

## Objetivo

Gerar documentacao tecnica e funcional a partir das evidencias encontradas no projeto.

## Instrucoes

1. Usar somente informacoes encontradas no codigo-fonte e na documentacao existente.
2. Gravar os documentos gerados em `/outputs`.
3. Preservar conteudo valido ja existente.
4. Documentar arquitetura, modulos, regras de negocio, APIs, banco de dados, telas, validacoes e permissoes.
5. Registrar evidencias com caminho do arquivo onde a informacao foi encontrada.
6. Marcar como "Ponto a validar" tudo que nao estiver comprovado no codigo.

# Gerar documentação do projeto

Com base no inventário gerado em `/outputs/relatorios/inventario-projeto.md`, crie a documentação funcional e técnica do projeto.

## Objetivo
Gerar documentação clara para desenvolvimento, QA, suporte e atendimento ao cliente.

## Saídas obrigatórias
Criar ou atualizar:

- `/docs/funcional/visao-geral.md`
- `/docs/funcional/modulos.md`
- `/docs/funcional/regras-negocio.md`
- `/docs/funcional/fluxos.md`
- `/docs/tecnica/arquitetura.md`
- `/docs/tecnica/configuracoes.md`
- `/docs/qa/cenarios-teste.md`
- `/docs/suporte/perguntas-frequentes.md`

## Cada módulo deve conter
- Nome
- Objetivo
- Usuários impactados
- Fluxo principal
- Regras de negócio
- Campos e validações
- Integrações
- Permissões
- Possíveis erros
- Cenários de teste sugeridos
