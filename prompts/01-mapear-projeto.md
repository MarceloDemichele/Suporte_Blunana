# Mapear Projeto

## Objetivo

Mapear o projeto a partir do codigo-fonte e da documentacao existente.

## Instrucoes

1. Ler os arquivos Markdown em `/docs`.
2. Ler os arquivos Markdown em `repositories/documentacao/docs/`, quando existirem.
3. Identificar arquitetura, modulos, fluxos, APIs, banco de dados, validacoes e permissoes.
4. Registrar evidencias com caminho do arquivo onde a informacao foi encontrada.
5. Marcar como "Ponto a validar" tudo que nao estiver comprovado no codigo.
6. Nao inventar regras, fluxos ou funcionalidades.

## Analise 

Analise este projeto e gere um inventário técnico e funcional.

Mapeie:
- estrutura de pastas
- módulos principais
- telas
- rotas
- serviços
- integrações
- regras de negócio aparentes
- arquivos de configuração
- documentação existente
- lacunas de documentação

Não altere código nesta etapa.
Salve o resultado em `/outputs/relatorios/inventario-projeto.md`.
